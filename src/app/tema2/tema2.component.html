Tema 2 - Modelos de Red

2.1 Arquitectura por niveles

2.1.1 Principios de la arquitectura por niveles

  • Primer principio: Si se quiere una comunicación bidireccional, cada nivel debe ser capaz de realizar tareas opuestas, una en cada sentido
  • Segundo principio: Los objetos bajo cada nivel en ambos sentidos deben ser idénticos

2.2 Conjunto de protocolos TCP/IP

Descripción de protocolos de red que se usa actualmente

2.2.1 Arquitectura de niveles

2.2.2 Niveles en el conjunto de protocolos TCP/IP

  1. Nivel Físico: Es responsable de transportar los bits individuales a través del enlace, la comunicación entre dos dispositivos es lógica. Los bits transportados son transformados a señales eléctricas u ópticas y enviados a través de los medios de transmisión (cable/aire)
  2. Nivel de enlace de datos: Puede haber varios conjuntos de de enlaces que se superpongan y que un datagrama pueda recorrer, los enrutadores son los responsables de elegir los mejores enlaces
  3. Nivel de red: Es el responsable de crear una conexión entre el ordenador de origen y el de destino, la comunicación en este nivel es deispositivo a dispositivo. Incluye el protocolo principal, el protocolo de internet (IP), que define el formato del paquete y de las direcciones utilizadas en este nivel.
    El protocolo IP no proporciona ningún control de flujo, errores o gestión.
  4. Nivel de transporte: El nivel de transporte en el host de origen recibe el mensaje de la capa de aplicación, lo encapsula en un paquete de nivel de transporte y lo envía al nivel de transporte en el host de destino. El nivel de transporte es responsable de dar servicios al nivel de aplicación. El principal es TCP
    El protocolo TCP proporciona control de flujo, de errores y de congestión. Otro protocolo famoso es el UDP, es como el TCP pero sin crear al principio una conexión lógica.
  5. Nivel de aplicación: Los niveles de aplicación de los dos extremos intercambian mensajes entre sí pasando antes por el resto de niveles.
    La comunicación en el nivel de aplicación es entre procesos, un proceso envía una solicitud al otro y recibe una respuesta. Los protocolos más famosos son HTTP, SMTP, FTP, SSH...

2.2.3 Encapsulado y desencapsulado

  • Encapsulado en el host fuente: En la fuente ssolo tenemos encapsulado.
    1. En el nivel de aplicación, los datos que se intercambian se denominan mensaje.
    2. El nivel de transporte toma el mensaje como carga útil y le añade el encabezado del nivel de transporte que contiene los identificadores de lo programas de aplicación de origen y destino. El resultado de mensaje + encabezado se llama segmento en TCP y datagrama de usuario en UDP, después de esto se transfiere a nivel de red.
    3. El nivel de red toma el paquete de transporte y le añade un encabezado propio que contiene las direcciones de los host de origen y destino. El resultado es el paquete de nivel de red que se transsfiere al nivel de enlace de datos.
    4. El nivel de enlace de datos toma el paquete del nivel de red y añade su propio encabezado, que contiene las direcciones del nivel de enlace del host o del siguiente salto(enrutador). El resultado es el paquete de nivel de enlace, que se denomina trama. La trama se envía al nivel físico.
  • Desencapsulado y encapsulado en el enrutador: En esta parte hay tanto encapsulado como desencapsulado ya que el enrutador conecta a dos o más enlaces.
    1. Tras recibir el conjunto de bits en el nivel de enlace de datos, este nivel desencapsula el datagrama de la trama y lo pasa a nivel de red
    2. El nivel de red solo inspecciona las direcciones de origen y destino en el encabezado del datagrama su consulta su tabla de reenvío para encontrar el siguiente salto al que debe entregar dicho datagrama. El datagrama se pasa entonces al nivel de enlace de datos del siguiente enlace.
    3. El nivel de enlace de datos del siguiente enlace encapsula el datagrama en una trama y lo pasa al nivel físico para su transmisión.
  • Desencapsulado en el host de destino: En el host de destino, cada nivel desencapsula el paquete recibido, coge la carga útil y la transmite al nivel superior hasta que el mensaje llega a la capa de aplicación (Implica comprobación de errores).
